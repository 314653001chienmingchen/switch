# 第二步：認識光模組（Optical Transceiver）

## 什麼是光模組？

光模組（Optical Transceiver）是一種安裝在交換機（Switch）、路由器（Router）、伺服器（Server）或網路卡（NIC）上的可插拔通訊模組。

它的外觀看起來很像一支金屬製的隨身碟，前端有光纖插孔，後端則是一排金屬接點，可以直接插入交換機的 Port。

雖然外型不大，但它負責的是整個高速網路中最重要的一件事情：

> **將交換機內部的電訊號（Electrical Signal）與光纖中的光訊號（Optical Signal）互相轉換。**

如果沒有光模組，交換機就無法透過光纖與其他設備建立高速連線。

---

# 為什麼需要光模組？

## 電訊號的限制

交換機內部所有 ASIC（交換晶片）、CPU、記憶體彼此之間都是利用**電訊號**溝通。

電訊號本質上就是：

- 高電壓 = 1
- 低電壓 = 0

也就是數位世界最基本的 Binary（0 與 1）。

例如：

```
1011001010110010
```

交換機晶片看到的就是這種高速切換的電壓。

但是電訊號有一個很大的缺點：

### 傳輸距離有限

如果使用銅線（Copper Cable）：

- 傳越遠訊號越弱（Signal Attenuation）
- 容易受到電磁干擾（EMI）
- 高速時容易失真
- 功耗增加

例如：

- 一條一般 RJ45 網路線最多約 100 公尺
- 若速度提升到 100G、400G，就更不適合長距離傳輸

因此，大型資料中心（Data Center）不可能全部都使用銅線。

---

# 光訊號的優勢

因此高速網路改採：

> **光纖（Fiber Optic Cable）**

光纖利用的是：

**雷射光（Laser）**

或

**LED 光源**

來傳送資料。

光訊號有許多優點：

- 可傳輸數公里甚至數十公里
- 幾乎沒有電磁干擾
- 頻寬極高
- 可支援 100G、400G、800G
- 延遲非常低
- 功耗相對較低

因此目前大型雲端資料中心（AWS、Google、Microsoft、Meta）幾乎全面使用光纖互連。

---

# 光模組的工作

既然：

交換機只能理解電訊號

但是

光纖只能傳送光訊號

那就需要一個中間翻譯器。

光模組就是：

> **Electrical ↔ Optical Converter**

也就是：

**電光轉換器（Electro-Optical Converter）**

它負責兩件事情：

## 1. 電轉光（Transmit）

當交換機送出資料時：

```
ASIC
   │
電訊號
   │
光模組
   │
轉成雷射光
   │
光纖
```

光模組會把：

```
010101010111010
```

轉換成：

```
亮
暗
亮
亮
暗
暗
亮
……
```

也就是利用雷射光快速閃爍來表示資料。

這一端稱為：

**Tx（Transmit）**

意思就是：

> 發射端

---

## 2. 光轉電（Receive）

另一端收到光後：

```
光纖
   │
光模組
   │
光轉電
   │
ASIC
```

光模組內部的光感測器（Photodiode）會把雷射光重新轉回電訊號。

例如：

```
亮 暗 亮 暗 亮
```

重新變成：

```
10101
```

交給交換晶片繼續處理。

這一端稱為：

**Rx（Receive）**

意思就是：

> 接收端

---

# 光模組的兩端

一個光模組通常可以分成兩個部分。

## 交換機端（Electrical Interface）

```
┌─────────────────┐
│                 │
│  金屬接腳        │
│                 │
└─────────────────┘
```

這一排金屬接腳插入交換機後：

- 接收 ASIC 的高速電訊號
- 把資料送回交換機
- 提供電源
- 與交換機進行管理通訊

因此它負責的是：

**Electrical Interface**

---

## 光纖端（Optical Interface）

另一端通常有兩個孔：

```
   ○   ○
```

分別是：

```
Tx      Rx
```

### Tx（Transmit）

負責：

把雷射光射出去

---

### Rx（Receive）

負責：

接收另一台設備送來的雷射光。

因此：

```
交換機A

Tx ─────────► Rx

Rx ◄───────── Tx

交換機B
```

所以兩條光纖通常是：

一條負責送

一條負責收

（部分 BiDi 光模組可利用不同波長，在同一條光纖上同時完成收發。）

---

# 光模組的內部有哪些元件？

一顆光模組其實是一台微型光通訊設備。

裡面通常包含：

```
┌────────────────────────┐
│ 控制晶片 MCU            │
│                        │
│ DSP                    │
│                        │
│ Laser Driver           │
│                        │
│ TIA 放大器             │
│                        │
│ Laser                  │
│                        │
│ Photodiode             │
│                        │
│ EEPROM                 │
└────────────────────────┘
```

主要元件功能如下：

| 元件 | 功能 |
|------|------|
| Laser | 發射雷射光 |
| Photodiode | 接收光並轉成電訊號 |
| Laser Driver | 控制雷射發光 |
| TIA（Transimpedance Amplifier） | 放大接收到的微弱光電流 |
| DSP（Digital Signal Processor） | 處理高速訊號、補償失真、錯誤校正 |
| MCU | 管理模組運作與監控狀態 |
| EEPROM | 儲存模組資訊（型號、廠商、序號、速率等） |

---

# 光模組的主流規格

光模組依據尺寸與速度有不同規格。

初學者不需要記住所有型號，只需要先建立「尺寸」與「速度」的概念即可。

---

## SFP 系列（Small Form-factor Pluggable）

外型：

像小型金屬隨身碟。

寬度約：

**1 公分左右**

常見型號：

- SFP
- SFP+
- SFP28

速度大約：

| 型號 | 常見速度 |
|------|----------|
| SFP | 1 Gbps |
| SFP+ | 10 Gbps |
| SFP28 | 25 Gbps |

通常應用於：

- Access Switch
- Enterprise Network
- Server 連線
- 低至中速資料中心

---

## QSFP 系列（Quad Small Form-factor Pluggable）

外觀看起來：

比 SFP 寬大約兩倍。

名稱中的：

**Q = Quad（四路）**

代表：

模組內部有四條高速傳輸通道（Lane）。

例如：

```
Lane 1

Lane 2

Lane 3

Lane 4
```

四條通道一起工作，因此速度可以大幅提升。

常見型號：

- QSFP+
- QSFP28
- QSFP56
- QSFP-DD

常見速度：

| 型號 | 常見速度 |
|------|----------|
| QSFP+ | 40 Gbps |
| QSFP28 | 100 Gbps |
| QSFP56 | 200 Gbps |
| QSFP-DD | 400 Gbps～800 Gbps |

目前大型 AI 資料中心幾乎都採用 QSFP-DD 或更新世代的高速光模組。

---

# SFP 與 QSFP 的比較

| 項目 | SFP 系列 | QSFP 系列 |
|------|-----------|------------|
| 外型 | 較小 | 較大 |
| 寬度 | 約 1 公分 | 約 2 公分 |
| 通道數 | 1 Lane | 4 Lane（或更多） |
| 常見速度 | 1G～25G | 40G～800G |
| 應用 | 一般企業網路 | 資料中心、高速交換機、AI 叢集 |

---

# 在資料中心中的角色

在現代資料中心中，一台交換機可能擁有數十甚至上百個高速光模組插槽。

例如：

```
┌──────────────────────────────┐
│ □ □ □ □ □ □ □ □ □ □ □ □ □ □ │
│ □ □ □ □ □ □ □ □ □ □ □ □ □ □ │
└──────────────────────────────┘
```

每一個插槽都可安裝一顆光模組，並透過光纖與其他交換機、伺服器或儲存設備連接。

在 AI 訓練叢集（AI Cluster）中，大量 GPU 之間需要極高速且低延遲的資料交換，因此高速光模組已成為不可或缺的核心元件。

---

# 重點整理

- 光模組（Optical Transceiver）負責在**電訊號**與**光訊號**之間進行雙向轉換。
- 交換機內部只能處理電訊號，因此需要光模組才能透過光纖進行高速通訊。
- 光模組包含兩個主要介面：靠近交換機的是電氣介面（Electrical Interface），靠近光纖的是光學介面（Optical Interface）。
- Tx（Transmit）負責發射光訊號；Rx（Receive）負責接收光訊號。
- 光模組內部包含 Laser、Photodiode、DSP、MCU、EEPROM 等元件，共同完成高速資料傳輸。
- SFP 系列體積較小，主要支援 1G～25G；QSFP 系列體積較大，透過多條通道（Lane）支援 40G～800G 等高速應用。
- 在現代雲端資料中心與 AI 叢集中，高速 QSFP 系列光模組已成為交換機之間互連的重要元件。
