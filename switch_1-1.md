# 光纖模組插槽（SFP / QSFP）介紹

光纖模組插槽（通常指 **SFP** 和 **QSFP** 插槽）是網路設備（如交換機、路由器、防火牆）上最常見的熱插拔介面，用於連接光纖或銅纜，以實現高速資料傳輸。

本文整理 SFP 與 QSFP 的核心差異、常見規格、相容性，以及模組選型建議。

---

# 核心差異：SFP vs QSFP

* TX（發射端，來自 Transmit）：負責將內部處理好的訊號轉換為光訊號（光波）發送出去。
* RX（接收端，來自 Receive）：負責捕捉外部傳來的光訊號，並將其轉換回設備可以處理的電子訊號。


| 項目 | SFP 系列 | QSFP 系列 |
|------|----------|-----------|
| 全名 | Small Form-factor Pluggable | Quad Small Form-factor Pluggable |
| 通道數 | 1 Tx + 1 Rx（單通道） | 4 Tx + 4 Rx（四通道） |
| 主要用途 | 1G / 10G / 25G / 50G | 40G / 100G / 200G / 400G+ |
| 體積 | 較小 | 較大 |
| Breakout | 不支援 | 可拆分成 4 個 SFP/SFP28 連接埠（依設備支援） |

> **重點：**
>
> - **SFP** 為單通道設計。
> - **QSFP** 為四通道設計，因此能提供更高總頻寬。
> - 一個 QSFP 連接埠可透過 **Breakout Cable** 分接成四個較低速的連接埠（需設備支援）。

---

# 常見插槽規格與速率

## 1. SFP 系列（單通道）

| 規格 | 單通道速率 | 調變方式 | 備註 |
|------|-----------|----------|------|
| SFP | 1 Gbps | NRZ | 最常見 Gigabit 模組 |
| SFP+ | 10 Gbps | NRZ | 外型與 SFP 相同 |
| SFP28 | 25 Gbps | NRZ | 外型與 SFP+ 相同 |
| SFP56 | 50 Gbps | PAM4 | 新一代高速單通道 |

---

## 2. QSFP 系列（四通道）

| 規格 | 通道 | 每通道速率 | 總速率 |
|------|------|-----------|---------|
| QSFP+ | 4 | 10 Gbps | 40 Gbps |
| QSFP28 | 4 | 25 Gbps | 100 Gbps |
| QSFP56 | 4 | 50 Gbps | 200 Gbps |
| QSFP-DD | 8 | 50 / 100 Gbps | 400 Gbps、800 Gbps |

> **QSFP-DD（Double Density）** 將通道數由 4 條提升至 8 條，因此可支援更高頻寬。

---

# 插槽向下相容性

一般而言，只要硬體設計允許，會遵循：

> **高速插槽可相容低速模組，但低速插槽不能使用高速模組。**

## 常見相容性

| 插槽 | 可使用模組 |
|------|-----------|
| SFP+ | SFP+、SFP |
| SFP28 | SFP28、SFP+、SFP |
| QSFP28 | QSFP28、QSFP+ |
| QSFP-DD | 視設備支援，可相容 QSFP56、QSFP28 |

### 注意事項

- 必須同時支援設備硬體與韌體。
- 不同廠商（Cisco、Juniper、HPE、Arista…）可能限制第三方模組。
- Breakout 功能需設備支援。

---

# 依傳輸距離選擇模組

插槽只是介面，實際仍需搭配光模組或纜線。

| 傳輸距離 | 建議媒介 | 常見模組 |
|-----------|----------|-----------|
| 1～7 公尺 | DAC（直連銅纜） | DAC Cable |
| 5～100 公尺 | AOC（主動式光纜） | AOC |
| 100～400 公尺 | 多模光纖（MMF） | SR、SR4 |
| 10～20 公里 | 單模光纖（SMF） | LR、LR4 |
| 40～80 公里 | 單模光纖（SMF） | ER、ZR |

---

# 光纖種類

## 多模光纖（MMF）

- 常見顏色：水藍色（Aqua）、橘色
- 成本較低
- 適合資料中心
- 傳輸距離較短
- 常搭配：
  - SR
  - SR4

---

## 單模光纖（SMF）

- 常見顏色：黃色
- 成本較高
- 適合長距離
- 可傳輸數十公里
- 常搭配：
  - LR
  - ER
  - ZR

---

# DAC、AOC 與光模組比較

| 類型 | 傳輸介質 | 距離 | 成本 | 是否可更換 |
|------|----------|------|------|-----------|
| DAC | 銅纜 | 1～7 m | ★ | 否 |
| AOC | 光纜 | 5～100 m | ★★ | 否 |
| 光模組 + MMF | 多模光纖 | 100～400 m | ★★★ | 是 |
| 光模組 + SMF | 單模光纖 | 10～80 km | ★★★★ | 是 |

---

| 速率 | 常見介面 |
|------|----------|
| 1G | SFP |
| 10G | SFP+ |
| 25G | SFP28 |
| 40G | QSFP+ |
| 50G | SFP56 |

| 800G | QSFP-DD（新世代） |

---

# 選型建議

### 企業網路

- 1G：SFP
- 10G：SFP+
- 25G：SFP28

---

### 資料中心

- 25G Server：SFP28
- 100G Spine：QSFP28
- 400G Backbone：QSFP-DD

---

### 距離選擇

- 同機櫃：DAC
- 同機房：AOC
- 數百公尺：SR + MMF
- 數公里以上：LR／ER + SMF

---

# 重點整理

- **SFP**：單通道，小型模組，最高可達 50G。
- **QSFP**：四通道，高頻寬，可支援 40G、100G、200G。
- **QSFP-DD**：八通道，可提供 400G、800G。
- 高速插槽通常可向下相容低速模組，但反向不成立。
- 短距離建議使用 DAC 或 AOC；中長距離則依需求選擇多模（MMF）或單模（SMF）光纖搭配相應模組。

---

# 管理與控制埠（Management Ports）

除了前方大量提供資料傳輸的網路埠（Ethernet Port）之外，交換機通常還會配置幾個專門供管理、設定及維護使用的介面。這些介面並不負責一般網路封包的傳輸，而是提供管理人員對設備進行初始設定、遠端維護及故障排除。

---

# 1. Console 埠（控制台埠）

Console 埠是所有網路設備最基本的管理介面，幾乎每一台交換機、路由器、防火牆都會配備。

## 功能

Console 埠主要提供**本地端（Local）管理**。

即使交換機尚未完成網路設定、IP 位址錯誤、管理介面故障，仍然可以透過 Console 埠直接登入設備，因此又稱為**Out-of-Band（帶外管理）**方式。

管理人員可透過 Console 埠：

- 初次安裝交換機
- 設定管理 IP
- 修改 VLAN
- 設定密碼
- 更新韌體
- 查看系統 Log
- 故障排除（Troubleshooting）
- 密碼遺失時進行 Recovery

---

## 連接方式

Console 埠無法像一般網路埠直接接上網路。

通常需要：

```
筆記型電腦
      │
USB-to-Console 線
      │
Console Port
      │
交換機
```

以前常見使用 **RJ45 Console Port**，需要一條特殊的 Console Cable。

現在許多新設備則直接提供：

- USB Mini-B
- USB Type-C
- Micro USB

方便直接與電腦連接。

---

## 使用軟體

連接完成後，需要使用 Terminal 軟體登入，例如：

- PuTTY（Windows）
- Tera Term
- SecureCRT
- MobaXterm
- minicom（Linux）

常見設定值：

| 項目 | 設定值 |
|------|--------|
| Baud Rate | 9600 bps |
| Data Bits | 8 |
| Stop Bit | 1 |
| Parity | None |
| Flow Control | None |

通常稱為 **9600 8N1**。

---

## Console 的特性

優點：

- 不需要 IP 位址
- 不需要網路
- 可在設備故障時使用
- 最安全的管理方式

缺點：

- 必須人在設備旁邊
- 無法遠端登入
- 需要專用 Console 線

---

# 2. Management 埠（MGMT Port）

Management Port（管理埠）是一個專門提供遠端管理的乙太網路介面。

外觀看起來與一般 RJ45 網路孔相同，但用途完全不同。

通常會標示：

- MGMT
- Management
- OOB
- ETH0

---

## 功能

Management Port 專門負責：

- 遠端登入設備
- 系統監控
- 韌體更新
- 備份設定
- 查看 Log
- SNMP 管理
- SSH 管理
- Web GUI 管理

它並不負責一般使用者的網路流量。

例如：

```
使用者電腦
      │
 Access Port
      │
──────────────
│  Switch     │
│             │
│  MGMT Port──管理網路
──────────────
```

管理流量與使用者資料流量互相分離。

---

## 為什麼需要獨立的 MGMT Port？

若直接使用一般 Ethernet Port 管理設備：

- VLAN 設定錯誤可能導致無法登入
- Routing 問題可能造成設備失聯
- 使用者流量可能影響管理流量

因此大型企業通常建立：

**Management Network（管理網路）**

所有交換機、路由器、防火牆的 MGMT Port 都接到同一個管理網路。

管理人員只需連入管理網路即可維護所有設備。

---

## Out-of-Band Management

MGMT Port 最大特色就是：

> **Out-of-Band Management（帶外管理）**

意思是：

管理流量與正式營運網路完全分開。

即使正式網路故障：

- VLAN 掛掉
- Routing 出問題
- STP 異常
- Link Down

管理人員仍可透過 MGMT Port 登入設備。

因此資料中心（Data Center）幾乎都採用帶外管理。

---

## Management Port 的優點

- 可遠端登入
- 管理流量獨立
- 安全性高
- 網路故障仍可管理
- 適合大型企業與資料中心

---

# Console 與 Management Port 比較

| 項目 | Console | Management Port |
|------|----------|-----------------|
| 是否需要 IP | 否 | 是 |
| 是否透過網路 | 否 | 是 |
| 是否可遠端 | 否 | 可以 |
| 是否受 VLAN 影響 | 否 | 不受正式 VLAN 影響（依管理網路設定） |
| 初始設定 | ✔ | ✘ |
| 故障排除 | ✔ | ✔ |
| 使用介面 | Serial / USB | Ethernet |

# 背面的電源與風扇

交換機通常全年無休運作，因此硬體設計十分重視可靠性與散熱能力。

大型企業或資料中心使用的交換機，背面通常配置電源模組（PSU）及風扇模組（Fan Module），兩者皆可熱插拔（Hot Swap）。

---

# 1. 電源供應器（Power Supply Unit，PSU）

PSU 的功能是將交流電（AC）或直流電（DC）轉換為交換機內部所需的各種電壓。

例如：

```
市電 AC 110V / 220V
        │
      PSU
        │
12V / 5V / 3.3V
        │
交換機主機板
```

---

## 雙電源備援（Redundant Power Supply）

資料中心交換機通常會安裝兩顆 PSU。

```
┌─────────────────────┐
│ PSU 1 │ PSU 2 │
└─────────────────────┘
```

正常情況下：

- 兩顆同時供電（Load Sharing）
- 或一顆供電、一顆待命（Standby）

若其中一顆故障：

```
PSU1 故障
     ↓
PSU2 立即接手供電
```

交換機不需重新開機，也不會中斷服務。

---

## 熱插拔（Hot Swap）

大多數企業級 PSU 都支援：

**Hot Swap（熱插拔）**

意思是：

不用關閉交換機即可：

- 拔除故障 PSU
- 插入新的 PSU

設備仍維持正常運作。

---

## AC 與 DC 電源

企業交換機通常支援兩種 PSU：

| 類型 | 使用場所 |
|------|-----------|
| AC Power Supply | 一般企業機房 |
| DC Power Supply | 電信機房、基地台 |

DC 電源常見於電信設備，能與大型 UPS 或直流供電系統整合。

---

# 2. 風扇模組（Fan Module）

高速交換機內部 ASIC 晶片功耗可高達數百瓦，因此需要強力散熱。

若散熱不足：

- CPU 過熱
- ASIC 降頻
- 網路效能下降
- 系統自動關機

因此企業級交換機通常配備多顆高速風扇。

---

## 模組化風扇

大型交換機通常採用可拆式風扇模組：

```
┌────────────────────────┐
│ FAN1 │ FAN2 │ FAN3 │
└────────────────────────┘
```

若某顆故障：

- 系統立即告警
- 可直接更換
- 不需關機

---

## 氣流方向（Airflow）

資料中心十分重視散熱方向，因此風扇通常分為：

### Front-to-Back

```
冷空氣
   ↓
[交換機]
   ↓
熱空氣
```

冷空氣由前方吸入，熱空氣由後方排出。

---

### Back-to-Front

```
冷空氣
   ↑
[交換機]
   ↑
熱空氣
```

依機櫃散熱設計而定，採相反方向散熱。

因此購買交換機時，需確認風扇氣流方向是否符合機房規劃。

---

## 噪音

企業級交換機風扇轉速通常介於 **8,000～20,000 RPM**，產生的噪音可達 **60～80 dB** 甚至更高。

因此：

- 一般辦公室不適合放置資料中心交換機。
- 機房通常設有獨立空調與隔音設備。
- 工程師進入機房時，長時間工作可能需配戴耳罩以降低噪音影響。

---

# 重點整理

- **Console 埠**：提供本地端管理，不需 IP 位址，適用於初始設定與故障排除。
- **Management（MGMT）埠**：提供遠端帶外管理，管理流量與正式網路分離，提高維護便利性與安全性。
- **PSU（電源供應器）**：企業級交換機多採雙電源備援與熱插拔設計，確保設備持續運作。
- **Fan Module（風扇模組）**：提供高效散熱，多數支援熱插拔，並需配合資料中心的氣流方向規劃，以維持最佳散熱效能。
